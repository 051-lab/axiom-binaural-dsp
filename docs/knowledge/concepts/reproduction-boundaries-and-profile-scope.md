# Reproduction Boundaries And Profile Scope

Type: concept note
Source IDs:

- `accurate-sound-reproduction-using-dsp`
- `the-audio-programming-book`
- `principles-and-applications-of-spatial-hearing`

## Purpose

This note helps distinguish Axiom Core enhancement work from future profile,
device, room, or correction work.

## Axiom Test-Design Use

Axiom Core should remain speaker-neutral unless a future accepted decision says
otherwise. Route-specific observations should be classified before they become
DSP hypotheses:

- Core: behavior expected to hold across ordinary routes;
- Reference: accuracy, translation, or correction-oriented profile work;
- Immersive: deliberate spatial presentation beyond Core neutrality;
- Night: low-level or fatigue-sensitive listening profile;
- Studio Path: monitoring, translation, or production workflow;
- Host/device: crossfeed, Android, Bluetooth, DAC, headphone, speaker, or room
  behavior that does not belong inside the Core EEL script.

## Axiom Questions

- Is this a Core defect or a route-specific preference?
- Does the evidence require a new profile rather than a new accepted baseline?
- Are device, room, or headphone assumptions leaking into Core?
- Should a finding become a Knowledge note, Labs experiment, or candidate
  hypothesis?

## Boundary

Profile scope prevents Axiom Core from absorbing every useful audio idea. A
route-specific improvement should not become a Core change without evidence
that it survives the Core qualification and listening gates.
