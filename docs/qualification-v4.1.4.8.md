# v4.1.4.8 Qualification Record

## Decision

`src/axiom_binaural_dsp_v4.1.4.8.eel` is the accepted Axiom listening
baseline as of 2026-05-25. It preserves `.7` at the default `+4 dB` Sub
Harmonics setting and adds terminal output reserve only above that setting.
No `.9` DSP iteration is justified by the completed `.8` checks.

The accepted host-owned terminal stage for this baseline is:

| Setting | Value |
|---------|-------|
| JDSP master processing | enabled |
| Limiter threshold | `-1.00 dB` |
| Limiter release | `60 ms` |
| Postgain | `0 dB` |
| Crossfeed and other host effects | disabled during comparison |

Crossfeed remains a playback choice in JamesDSP and is not part of the
measured Axiom core.

## Deterministic Host Qualification

The managed WSL workflow captured `.7` and `.8` through the live JDSP host
path while holding the limiter threshold at `-1.00 dB` for the full test
session. The suite completed with `PASS`.

| Check | Result | Observation |
|-------|--------|-------------|
| Default bass burst transparency | PASS | `.8` delta `-0.002 dB` |
| Default sustained 90 Hz pressure transparency | PASS | `.8` delta `-0.000 dB`, peak `-0.995 dBFS` |
| Default sweep transparency | PASS | `.8` delta `+0.003 dB` |
| Default correlated mono transparency | PASS | `.8` delta `+0.064 dB` |
| Default side-only transparency | PASS | `.8` delta `+0.002 dB` |
| `+8 dB` bass burst reserve | PASS | `.8` delta `-4.001 dB` |
| `+8 dB` correlated mono reserve | PASS | `.8` delta `-3.902 dB` |
| `+8 dB` side-only reserve | PASS | `.8` delta `-3.970 dB` |
| `+8 dB` sustained pressure margin | PASS | `.8` peak `-1.563 dBFS` |
| `+12 dB` boundary margin | PASS | `.8` supplied `5.397 dB` additional margin |
| Three-passage generated program corpus | PASS | integrity and margin checks passed |

The program corpus is generated by `scripts/generate_axiom_program_corpus.py`.
It is deterministic engineering stimulus material, not a substitute for
listening evaluation.

## Repeated Accepted-Setting Stress Baseline

A subsequent host-path baseline measurement repeated each registered external
excerpt three times through accepted `.8` at the tracked `-1.00 dB` limiter
threshold. All twelve captures were unclipped and supplied repeatable level
metrics, establishing a dense-material reference for later candidates.

| Excerpt | Mean peak (dBFS) | Highest repeated peak (dBFS) | Clipped samples | Result |
|---------|-----------------:|-----------------------------:|----------------:|--------|
| Electronic / high-energy | `-0.606` | `-0.542` | `0` | PASS |
| Instrumental hip hop | `-0.757` | `-0.706` | `0` | PASS |
| Jazz/fusion low-end passage | `-0.982` | `-0.980` | `0` | PASS |
| Orchestral passage | `-0.917` | `-0.881` | `0` | PASS |

This result retains `-1.00 dB` as the qualified host threshold. A separate
diagnostic run at `-0.50 dB` clipped the electronic excerpt and is therefore
not acceptable as the host baseline.

## Listening Acceptance

After the measurement checks, device listening on multiple songs confirmed no
mute, distortion, clipping symptom, or objectionable tonal regression in
`.8`. That human listening result is the acceptance gate for the baseline;
the host captures are regression and headroom evidence, not a complete measure
of subjective audio quality.

## Reproduce On The WSL Workstation

Run the managed capture path:

```bash
scripts/run_jdsp_wsl_qualification.py \
  src/axiom_binaural_dsp_v4.1.4.7.eel \
  src/axiom_binaural_dsp_v4.1.4.8.eel \
  /tmp/axiom-v47-v48-wsl-qualification
```

To add locally controlled listening material, supply a manifest outside this
repository using `--local-material-manifest /absolute/path/to/manifest.json`.
The managed runner sets and restores the terminal-limiter threshold once for
the session. This avoids treating asynchronous host configuration changes as
script behavior.

## Boundary For Next Work

Changes after `.8` should improve measurement confidence, documentation, or a
demonstrated audio weakness. A new EEL version should not be introduced solely
because another parameter or DSP stage is available.
