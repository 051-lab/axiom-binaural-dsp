Signal Chain

What the audio engine does.

The accepted chain is a controlled mastering-style enhancement path. Each stage has an ownership boundary so the script does not fight the host limiter, host crossfeed, or later device-specific playback decisions.

Input DC protection

15 Hz high-pass protection before nonlinear and dynamic stages.

Spatial 3-way M/S field

Bass below 150 Hz folds toward mono; low-mid and high side width stay independent.

Bass Additive harmonics

90 Hz extraction, soft saturation, harmonic isolation, and additive injection.

Air Dynamic exciter

Level-contingent high-frequency enhancement above 11 kHz with sample-rate timing.

Control STFT suppressor

Stereo-linked per-bin attenuation in the 2-6 kHz resonance region.

Output Reserve and host limiter

Fixed -1 dB reserve plus bass-aware reserve above +4 dB Sub Harmonics.

Architecture

The current baseline is intentionally restrained.

The major improvements have been subtractive or ownership-based: removing internal limiter and crossfeed responsibilities, removing redundant dry-path filtering, then tightening output reserve and low-mid width with real-host evidence.

Engineering System

The repo now contains a gated DSP lab.

The Pi engineering harness, real-JDSP capture tools, static validation, local material manifest, and GitHub publication workflow are now part of how Axiom changes are made.

Controlled candidate flow

  • 1Open an investigation anchored to the accepted baseline hash.
  • 2Record a falsifiable hypothesis and a listening target before mutation.
  • 3Create a versioned external worktree for sound-changing EEL work.
  • 4Run static, unit, and serialized real-JDSP qualification gates.
  • 5Require explicit human listening, publication, and merge approval.

Tooling built around Axiom

  • AStatic EEL safety checks for reserved constants, STFT signatures, memory layout, and final output writes.
  • BReal-host render path through JDSP, Pulse routing, limiter policy, and repeatability qualification.
  • CSpecialized screens for reserve law, width behavior, STFT behavior, dynamic exciter sensitivity, stage observability, and low-level exciter probes.
  • DDocumentation records that explain why a candidate was accepted or why an idea was closed.
  • EGitHub PR workflow for draft publication, explicit approval, merge, and post-merge verification.
