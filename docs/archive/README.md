# Axiom Archive

This folder keeps completed, historical, or reference material out of the
active documentation path without deleting project evidence.

## Folders

| Folder | Contents |
| --- | --- |
| `labs/` | Completed Labs plans, listening targets, summaries, and decision maps. |
| `planning/` | Older ecosystem, agentic, and system-roadmap planning records. |
| `reviews/` | Full-system reviews, progression reviews, and audit standstill records. |
| `local-maps/` | Historical local-directory maps and cleanup notes. |

## Rule

Archive means "not active guidance." It does not mean the record is invalid.
Use archived docs as evidence or history, but start new work from:

- `AXIOM.md`;
- `docs/README.md`;
- `docs/dsp-change-workflow.md`;
- `docs/system-status.md`;
- `docs/release-gates.md`.
